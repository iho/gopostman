# gopostman

A CLI code generator that converts a Postman collection into Go integration tests for a [testify](https://github.com/stretchr/testify) suite.

## Features

- Parses Postman collection v2.1 JSON
- Sorts folders and requests alphabetically at every nesting level
- Generates one test method per request as a method on your suite struct
- Injects `Authorization: Bearer <s.token>` on every request
- Passes request bodies (`raw` mode) via `strings.NewReader`
- Skips disabled headers; skips any `Authorization` header already present in the collection
- Handles both string and object URL formats from Postman
- Uses [jennifer](https://github.com/dave/jennifer) for clean, properly formatted Go output

## Installation

```sh
go install github.com/iho/gopostman@latest
```

Or build from source:

```sh
git clone https://github.com/iho/gopostman
cd gopostman
go build -o gopostman .
```

## Usage

```
gopostman <collection.json> <output.go> [suite-name] [package-name]
```

| Argument         | Required | Default                        | Description                                      |
|------------------|----------|--------------------------------|--------------------------------------------------|
| `collection.json`| yes      | —                              | Path to the Postman collection v2.1 JSON file    |
| `output.go`      | yes      | —                              | Destination `.go` file (created or overwritten)  |
| `suite-name`     | no       | `Suite`                        | Name of the testify suite struct                 |
| `package-name`   | no       | directory name of `output.go`  | Go package name for the output file              |

### Example

```sh
gopostman collection.json ./tests/api_gen_test.go Suite integration_test
```

## Generated output

Given a collection with two folders (`Auth`, `Users`) and several requests, the generator produces:

```go
// Code generated by gopostman. DO NOT EDIT.
package integration_test

import (
    "net/http"
    "strings"
)

func (s *Suite) TestAuth_Login() {
    req, err := http.NewRequest("POST", "https://api.example.com/v1/auth/login", strings.NewReader(`{"email":"user@example.com","password":"secret"}`))
    s.Require().NoError(err)
    req.Header.Set("Authorization", "Bearer "+s.token)
    req.Header.Set("Content-Type", "application/json")
    resp, err := http.DefaultClient.Do(req)
    s.Require().NoError(err)
    defer resp.Body.Close()
    s.Equal(http.StatusOK, resp.StatusCode)
}

func (s *Suite) TestUsers_GetAllUsers() {
    req, err := http.NewRequest("GET", "https://api.example.com/v1/users", nil)
    s.Require().NoError(err)
    req.Header.Set("Authorization", "Bearer "+s.token)
    req.Header.Set("Accept", "application/json")
    resp, err := http.DefaultClient.Do(req)
    s.Require().NoError(err)
    defer resp.Body.Close()
    s.Equal(http.StatusOK, resp.StatusCode)
}
```

Method names follow the pattern `Test<Folder>_<Request>`, with each segment title-cased and non-alphanumeric characters stripped.

## Suite setup

The generated file expects a testify suite struct with a `token` string field. A minimal setup looks like:

```go
package integration_test

import (
    "testing"
    "github.com/stretchr/testify/suite"
)

type Suite struct {
    suite.Suite
    token string
}

func (s *Suite) SetupSuite() {
    s.token = "your-bearer-token" // or fetch from env / login endpoint
}

func TestSuite(t *testing.T) {
    suite.Run(t, new(Suite))
}
```

## Postman variables

URLs containing Postman variable syntax (`{{variableName}}`) are preserved as-is in the generated string literals. Replace them manually or resolve them in `SetupSuite` / `SetupTest` by overriding the request URL after generation.
